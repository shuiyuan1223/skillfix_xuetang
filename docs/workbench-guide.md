# 技能调试工作台使用指南

## 概述

技能调试工作台（Skill Debug Workbench）是 PHA 的开发调试工具，提供独立的沙盒环境来测试和迭代健康 Skills 和解读 Prompts，**不影响主 Agent 运行时**。

## 访问入口

侧边栏直接点击 **"调试工作台"**（工作台图标）。若侧边栏未显示，检查 `.pha/config.json` 中 `whitelist` 是否覆盖了当前用户。

## 界面布局

三栏式布局：

```
┌──────────────┬──────────────────────┬────────────────────────────┐
│  测试数据     │   技能 / 提示词       │   运行结果                  │
│              │                      │                            │
│  JSON 编辑器  │  [技能] [提示词] 标签  │  模型选择 + [对比差异] [运行] │
│  字符计数     │  技能/提示词 列表      │                            │
│  清空按钮     │  代码编辑器           │  结果卡片（最多3条）         │
│              │  保存/回退 按钮        │                            │
│              │                      │  状态栏                     │
└──────────────┴──────────────────────┴────────────────────────────┘
```

## 使用流程

### 1. 准备测试数据

在左侧"测试数据"区域输入 JSON 格式健康数据或自然语言描述：

```json
{
  "sleep": { "duration": 420, "deep_sleep": 90, "rem": 60 },
  "heart_rate": { "resting": 65, "max": 172 },
  "steps": 8200
}
```

- **编辑/预览** 切换按钮可在代码编辑和 Markdown 预览间切换
- 右上角显示字符计数
- **清空** 按钮清除全部数据

### 2. 配置技能（Skills 标签页）

**技能列表**：
- 点击"状态"列徽章 → 切换该技能的启用/禁用
- 点击行 → 在下方代码编辑器中打开 `SKILL.md`
- **全部启用 / 全部禁用**：批量切换
- **展开/折叠**：控制列表高度

**编辑技能**：
1. 点击技能行选中
2. 在代码编辑器中修改 `SKILL.md` 内容
3. 保存按钮变为可用 → 点击**保存**写入磁盘（`.pha/workbench/skills/{id}/SKILL.md`）
4. **回退**：撤销本次未保存的编辑，恢复到上次保存（磁盘）的内容

> **版本说明**：工作台打开时会记录每个 Skill 的"基线版本"（`baselineContent`）。
> - **保存**：将当前编辑写入磁盘，`dirty` 标记清除
> - **回退**：恢复到上次保存的磁盘版本（按钮在 `dirty=false` 时禁用）
> - **基线版本**：工作台初始化时的快照，用于"对比差异"。刷新页面会重置基线。

### 3. 配置提示词（Prompts 标签页）

操作与技能相同。提示词文件位于 `.pha/workbench/prompts/{id}.md`。

默认包含 `health_interpretation` 提示词（健康数据综合解读模板）。

### 4. 运行解读

右侧栏操作区：

#### 4.1 选择模型

点击模型按钮选择：
- **GLM-5** — 默认
- **Kimi K2.5**
- **DeepSeek V3.2**

#### 4.2 普通解读

确保"对比差异"开关为**关闭**（灰色），点击 **运行解读**：

1. 自动组合：系统指令 + 启用的提示词 + 启用技能的 `<skill_guides>` + `<user_health_data>`
2. 调用选中模型，流式输出结果
3. 结果卡片显示：时间戳、模型、提示词/技能数、Token、耗时

#### 4.3 对比差异模式（语义高亮）

用于验证 Skill/Prompt 修改是否达到预期效果。

**开启方式**：点击 **"对比差异"** 按钮（蓝色高亮 = 已开启）→ 再点击 **运行解读**。

**工作原理**（固定使用 GLM-5，共 3 次 LLM 调用）：

```
步骤 1：Before  — 用基线版本（工作台打开时的快照）生成解读
步骤 2：After   — 用当前版本（你编辑后的内容）生成解读
步骤 3：分析    — GLM-5 理解 Skill/Prompt 的语义变化，
                  在 Before 和 After 输出中用 **粗体** 标注
                  受变更影响的关键句子/段落
```

**结果展示**：
- **变更摘要**：2-3 句话说明本次修改在语义上改了什么规则/行为，以及对输出的影响
- **Skill/Prompt 变更 Diff**（可折叠）：逐文件显示 unified diff
- **解读对比（语义标注）**：Before 和 After 两个解读全文，受影响段落用 **粗体** 标注

> **注意**：
> - 只对比"已启用"的 Skill/Prompt 中发生了变化的部分
> - 若修改的项未启用，系统会提示先启用
> - "基线版本"为工作台本次初始化时的内容快照，刷新页面后重置
> - 对比差异模式不受顶部模型选择影响（固定 GLM-5）

#### 4.4 保存后为何 Skill Diff 能检测到？

工作台使用 **磁盘上保存的内容** 与**基线内容**做 diff，而不是依赖编辑器中的临时状态：

- 只要你修改后保存（即磁盘内容变了），基线 vs 磁盘 = 有差异 → diff 可用 ✅
- 若只是在编辑器里改了但未保存 → 基线 vs editedContent = 有差异 → diff 也可用 ✅
- 若修改并保存后再点"回退" → editedContent 恢复为磁盘内容 → 与基线相比仍有差异 ✅

### 5. 结果卡片

每次运行生成一张结果卡片（最多保留 3 条）：

- **渲染/源码** 切换：在 Markdown 渲染和原始文本间切换
- **复制 Messages**：复制完整的输入 Prompt，方便在其他工具测试
- **对比差异结果**额外提供"复制旧 Messages"和"复制新 Messages"

### 6. 导出 ZIP

点击右上角 **下载 ZIP** 按钮，打包下载所有技能和提示词的当前内容：

```
workbench-export-{timestamp}.zip
├── skills/
│   ├── sleep-coach/SKILL.md
│   ├── heart-monitor/SKILL.md
│   └── ...
└── prompts/
    └── health_interpretation.md
```

## 将调试结果应用到主系统

1. 在工作台中迭代，直到效果满意
2. 点击"下载 ZIP"导出
3. 手动将文件复制到 `src/skills/{skill-id}/SKILL.md`
4. 运行 `bun run build && pha restart`

## 常见问题

### Q: 对比差异时提示"未检测到修改"？

A: 系统通过比较"基线版本"（工作台打开时的快照）和"当前版本"来判断是否有修改。原因可能是：
1. 你还没有对任何已启用的 Skill/Prompt 进行编辑（或编辑后又回退了）
2. 刷新了页面导致基线重置，与磁盘内容一致

### Q: 对比差异时提示"修改项未启用"？

A: 你修改了某个 Skill/Prompt，但它当前是禁用状态。请在技能列表中点击状态列将其启用，再运行对比。

### Q: 解读对比的高亮（粗体）不准确？

A: 语义标注由 GLM-5 根据理解自动生成，准确度取决于修改内容的清晰度。如果修改较小或语义不明显，LLM 可能标注不精确。可在对比结果的"变更摘要"中查看 LLM 对修改语义的理解。

### Q: 运行中提示"No prompt or test data"？

A: 确保以下至少一项成立：
- 左侧测试数据不为空
- 有已启用的提示词（内容非空）

### Q: 保存后回退到哪个版本？

A: "回退"恢复到**上次保存到磁盘的版本**（即最后一次点击"保存"的内容）。若刚刚保存后想撤销，需要手动重新编辑或使用基线对比找回差异。

## 技术细节

| 项目 | 说明 |
|------|------|
| 技能存储 | `.pha/workbench/skills/{id}/SKILL.md` |
| 提示词存储 | `.pha/workbench/prompts/{id}.md` |
| 状态管理 | 内存 `GatewaySession.workbenchState`，页面刷新重置 |
| Diff 计算 | `git diff --no-index`（`spawnSync`，Windows 无弹窗） |
| 流式输出 | SSE delta patch，避免大页面全量重建 |
| 入口文件 | `src/gateway/workbench-page.ts` |
| Action 处理 | `src/gateway/workbench-handlers.ts` |
| 初始化 | `src/gateway/workbench-init.ts` |
