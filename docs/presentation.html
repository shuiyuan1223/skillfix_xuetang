A2UI 协议 · 60+ 组件

Agent-to-UI — 服务端生成组件树，前端纯渲染。4 个 Surface：main / sidebar / modal / toast。涵盖布局、图表、表单、Diff、Git 时间线等。

JSON-RPC 2.0 Streamable HTTP。健康数据、Git 操作、记忆搜索、技能管理、计划与提醒等全域工具。

Skills 系统 · 20+

SKILL.md 格式的可热插拔专家知识。自动触发、UI 可视化编辑、Git 追踪。覆盖睡眠、心脏、营养、进化等领域。

双形态前端 + 多 Provider

Web UI (React) 和 TUI (终端) 消费同一套 A2UI 数据。统一模型仓库支持 9 家 LLM Provider。

核心设计原则

原则 1

前端无业务逻辑

Web UI 和 TUI 只接收 A2UI 消息并渲染。23+ 页面由服务端 A2UIGenerator 生成，前端做纯粹的 Component → DOM 映射。

// 服务端 const page = generateDashboardPage(data); send({ type: "page", surfaces: { main: page } }); // 前端 handleMessage(msg) → render(msg.surfaces.main);
原则 2

MCP + Skills 双轨能力

数据获取和操作执行通过 MCP Tools 暴露（JSON-RPC 标准）。专家知识和行为指导通过 Skills 热加载（Markdown 文件）。

// MCP Tool (原子能力) get_heart_rate({ date: "2026-02-18" }) → { avg: 72, max: 128, min: 49 } // Skill (解读框架) sleep-coach/SKILL.md → 睡眠分析 + 建议
原则 3

Git-Backed 状态管理

用户画像、记忆、技能均以 Markdown/JSON 文件存储，通过 Git 追踪变更。Worktree 隔离确保安全修改，任何改动可回溯、可 diff、可合并。

原则 4

Agent 自主进化

Evolution Lab 提供 SHARP 5 维评估框架。System Agent 自主诊断弱项、在 Worktree 分支修改代码、展示 Diff、验证改进效果 — 无需人工审批。

功能展示：对话式健康助手

自然语言交互

无需翻阅复杂界面，直接对话获取健康建议。Agent 自动调用 MCP 工具获取最新数据作为回答依据。

个性化记忆

Agent 记住用户的健康目标、偏好、历史对话。每次交互都建立在之前的上下文之上，向量索引支持语义搜索。

技能自动触发

用户消息匹配 triggers 后自动注入对应 Skill（睡眠教练、心脏监护、营养指导等），提供专业领域解答。

安全边界

内置医疗边界意识、风险披露、有害内容过滤等安全保障，确保建议在安全范围内。

功能展示：健康仪表盘

综合健康概览

Apple Watch 风格的活动环展示步数、活动时间、卡路里三大核心指标的完成进度。

系统 Agent：自主运维智能体

双 Agent 架构

PHA 用户 Agent 负责面向用户的健康对话；System Agent (SA) 负责系统内部运维 — 基准测试、提示词优化、版本管理。两者独立 Profile 配置。

SA 自主能力

  • 运行基准测试并分析评分维度
  • 在 Worktree 中通过 Claude Code 修改代码
  • 自主诊断弱项并直接应用修复
  • 展示 Diff 结果（可视化 diff_view 组件）
  • 重跑验证、对比效果后合并到 main
System Agent = 自主进化引擎
Worktree 隔离 → 自主修改 → Diff 展示 → 验证合并
