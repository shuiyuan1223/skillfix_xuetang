A2UI 协议

Agent-to-UI — 服务端生成组件树，前端纯渲染。4 个 Surface：main / sidebar / modal / toast

MCP 工具

JSON-RPC 2.0 标准协议。健康数据、Git 操作、技能管理等 30+ 工具

Skills 系统

SKILL.md 格式的可热插拔专家知识。睡眠教练、运动指导、基准评估等

双形态前端

Web UI (React) 和 TUI (终端) 消费同一套 A2UI 数据，零额外代码

核心设计原则

原则 1

前端无业务逻辑

Web UI 和 TUI 只接收 A2UI 消息并渲染。所有页面由服务端 A2UIGenerator 生成，前端做纯粹的 Component → DOM 映射。

// 服务端 const page = generateDashboardPage(data); send({ type: "page", surfaces: { main: page } }); // 前端 handleMessage(msg) → render(msg.surfaces.main);
原则 2

MCP + Skills 双轨能力

数据获取和操作执行通过 MCP Tools 暴露（JSON-RPC 标准）。专家知识和行为指导通过 Skills 热加载（Markdown 文件）。

// MCP Tool (原子能力) get_heart_rate({ date: "2026-02-18" }) → { avg: 72, max: 128, min: 49 } // Skill (解读框架) sleep-coach/SKILL.md → 睡眠分析 + 建议
原则 3

Git-Backed 状态管理

用户画像、记忆、技能均以 Markdown/JSON 文件存储，通过 Git 追踪变更。任何修改都有完整历史，可回溯、可 diff、可合并。

原则 4

Agent 自我进化

Evolution Lab 提供完整的基准测试框架（SHARP 5 维评估），Agent 可自主分析弱项、修改自身提示词和技能、验证改进效果。

功能展示：对话式健康助手

自然语言交互

无需翻阅复杂界面，直接对话获取健康建议。Agent 自动调用 MCP 工具获取最新数据作为回答依据。

个性化记忆

Agent 记住用户的健康目标、偏好、历史对话。每次交互都建立在之前的上下文之上。

计划生成与跟踪

根据用户需求自动创建减重计划、睡眠改善计划等，设定阶段目标和里程碑，定期提醒和复盘。

安全边界

内置医疗边界意识、风险披露、有害内容过滤等安全保障，确保建议在安全范围内。

进化流水线：6 步自我改进

1. Benchmark — 基准测试
55 个测试用例覆盖安全、有用性、准确性、相关性、个性化 5 大类 16 细项
2. Diagnose — 诊断弱项
分析失败用例，识别评分最低的维度和具体准则
3. Propose — 提出方案
System Agent 基于诊断结果，自动提出 SOUL.md / SKILL.md 修改方案
4. Approve — 人类审核
变更在 Git 分支隔离，Merge 前需用户确认（变更文件、Diff 预览）
5. Apply — 应用变更
Git worktree 分支管理，合并到 main 后立即生效
6. Validate — 验证效果
重跑基准测试，对比分数变化，确认改进有效
