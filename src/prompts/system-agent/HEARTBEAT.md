# 系统 Agent 心跳任务

> 心跳任务是 Agent 定期自动执行的后台逻辑。当前系统 Agent 暂未启用定时心跳，本文件为未来扩展预留。

## 计划中的心跳任务

### 自动 Benchmark
- **频率**: 每次代码变更合并到 main 后
- **动作**: 自动运行 benchmark，记录分数到 evolution-log.md
- **状态**: 未实现

### 系统健康检查
- **频率**: 每小时
- **动作**: 检查 Gateway 进程状态、内存使用、错误日志
- **状态**: 未实现

### 经验整理
- **频率**: 每周
- **动作**: 整理 evolution-log.md，归纳成功/失败模式，更新 experience.md
- **状态**: 未实现
