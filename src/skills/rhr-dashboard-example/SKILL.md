71）\n- **基线（71 bpm）**：靛色（#6366f1），标记起始点\n- **目标（60 bpm）**：绿色（#10b981），标记终点\n- **进度（60%）**：紫色（#8b5cf6），快速感知完成度\n\n**渲染效果**：4 列卡片行，最紧凑的信息密度。用户打开 dashboard 立即看到进度。\n\n### Section 2: 进度追踪 — progress_tracker\n\n**这是最关键的 widget！** 展示下降型目标的关键配置：\n\n```json\n{\n  \"type\": \"progress_tracker\",\n  \"config\": {\n    \"title\": \"RHR 下降进度\",\n    \"baseline\": 71,      // 必须设置！\n    \"current\": 65,\n    \"target\": 60,\n    \"unit\": \" bpm\",\n    \"color\": \"#8b5cf6\"\n  }\n}\n```\n\n**为什么 baseline 关键**：\n- 进度计算 = (baseline - current) / (baseline - target)\n- = (71 - 65) / (71 - 60) = 6 / 11 ≈ **54.5%**（约一半完成）\n- 如果不设 baseline，系统会当作\"升高型目标\"，进度算法错误！\n\n**下降型目标的三种常见场景**：\n1. **RHR 降低** ← 本示例\n2. **体脂率降低**\n3. **血压/血糖降低**\n\n### Section 3: 趋势分析\n\n**两张图结合**：\n\n#### 3.1 current) / (baseline - target) * 100\n    \n    # 构建 dashboard 配置\n    dashboard_config = {\n        \"title\": \"静息心率改善计划\",\n        \"sections\": [\n            section_core_stats(current, baseline, target, progress_pct),\n            section_progress_tracker(baseline, current, target),\n            section_trends(trends, weekly_data),\n            section_training_correlation(workouts, weekly_data),\n            section_roadmap(profile, current, target),\n            section_insights(trends, weekly_data, profile)\n        ]\n    }\n    \n    # 创建 dashboard\n    dashboard_id = create_dashboard(dashboard_config)\n    \n    # 生成解说文案\n    narrative = generate_narrative(\n        \"你的 RHR 改善计划正在稳步推进...\",\n        current, baseline, target, progress_pct\n    )\n    \n    return {\n        \"text\": narrative,\n        \"dashboard_id\": dashboard_id,\n        \"dashboard_url\": f\"/dashboards/{dashboard_id}\"\n    }\n```\n\n---\n\n## 最佳实践总结\n\n### ✅ DO\n- ✅ 总是设置 `baseline`（下降型目标必须）\n- ✅ 在 stat_row 中显示\"趋势\"（-6 bpm）\n- ✅ 用 line_chart 展示完整趋势，避免单点数据\n- ✅ 用 data_table 展示训练量与 RHR 的关联\n- ✅ 用 text_block 提供可操作的洞察和建议\n- ✅ 定期刷新（每天 1 次，异常时立即刷新）\n\n### ❌ DON'T\n- ❌ 不要用 progress_tracker 来展示升高型指标（如体重）——会算反\n- ❌ 不要显示过于频繁的更新（>= 1 小时刷新一次没有意义）\n- ❌ 不要在没有数据的情况下编造数值\n- ❌ 不要用太多 widget（7-8 个 widget 足够，避免信息过载）\n- ❌ 不要忽视时间窗口——RHR 趋势需要最少 4-8 周的数据才能看出规律\n\n---\n\n## 在 evolution-driver 中的应用\n\n如果要用 RHR Dashboard 测试 PHA Agent 的\"健康教练\"能力：\n\n1. **创建基准测试用例**：\n   ```\n   query: \"帮我创建一个 RHR Dashboard，追踪我的心肺改善\"\n   expected_dashboard:\n     -
