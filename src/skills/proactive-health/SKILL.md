# 主动健康管理

## 核心理念

PHA 不是被动问答工具，而是主动的健康管理助手。通过推荐、提醒、日历三大能力，在恰当时机主动干预用户的健康行为。

## 三大主动能力

### 1. 推荐（Recommendations）

**类似小艺建议 — 基于数据和上下文的主动推送。**

| 类型 | 触发时机 | 示例 |
|------|---------|------|
| `health_tip` | 发现数据异常或可优化点 | "你这周平均睡眠只有 6.1h，试试 22:30 设个就寝闹钟" |
| `plan_reminder` | 计划进度落后 | "你的步数计划本周完成 45%，今天多走 2000 步就能追上" |
| `achievement` | 达成目标或里程碑 | "连续 5 天睡眠超过 7 小时！保持住 💪" |
| `alert` | 健康指标异常 | "今天静息心率偏高 (92bpm)，注意休息" |
| `suggestion` | 一般性建议 | "明天天气适合户外运动，考虑下午去跑步？" |

**创建推荐的时机：**
- 分析健康数据后发现值得关注的点
- 计划进度检查时发现落后或超前
- 用户达成某个里程碑时
- 周回顾中发现持续趋势

**推荐优先级：**
- `urgent` — 健康警示（心率异常、血压偏高）
- `high` — 计划严重落后、重要里程碑
- `medium` — 日常健康建议、进度提醒
- `low` — 一般性 tips、鼓励

### 2. 提醒（Reminders）

**像系统待办 — 定时、可重复、可分类。**

| 类别 | 场景 | 重复规则 |
|------|------|---------|
| `sleep` | 就寝提醒 | daily |
| `exercise` | 运动计划 | weekdays / weekly |
| `medication` | 用药提醒 | daily |
| `hydration` | 喝水提醒 | daily (可多次) |
| `meal` | 饮食提醒 | daily |
| `checkup` | 体检预约 | none |

**创建提醒的时机：**
- 用户创建健康计划时 → 自动关联提醒
- 用户明确要求 "提醒我..."
- 发现用户作息不规律 → 建议设定固定提醒
- 用药场景 → 每日定时提醒

**提醒时间设定原则：**
- 就寝提醒：比目标入睡时间早 30 分钟
- 运动提醒：根据用户偏好时段（早/晚）
- 用药提醒：遵医嘱时间
- 喝水提醒：每隔 2-3 小时

### 3. 日历事件（Calendar Events）

**像系统日历 — 有时间跨度、可管理。**

| 类别 | 场景 | 示例 |
|------|------|------|
| `workout` | 运动安排 | "周三 18:00-19:00 跑步" |
| `checkup` | 体检预约 | "3月15日 上午 体检" |
| `meal` | 饮食安排 | "周日 meal prep 时间" |
| `sleep` | 作息计划 | "22:30 开始晚间放松" |
| `medication` | 复诊安排 | "下周二 心内科复诊" |

**创建日历事件的时机：**
- 用户安排具体活动时间
- 健康计划中有时间安排的部分
- 体检、复诊等一次性事件
- 周计划制定时

## 与健康计划联动

**创建计划时自动设置：**

```
用户: "帮我制定一个睡眠改善计划"
Agent:
1. 创建 health_plan (sleep improvement)
2. 创建 reminder: "晚间放松 — 放下手机，准备入睡" (daily, 22:30)
3. 创建 reminder: "记录今日睡眠感受" (daily, 08:00)
4. 创建 recommendation: "你的睡眠计划已启动，前三天最重要..."
```

**进度落后时主动推荐：**

```
系统检测到步数计划进度 < 60%:
→ create_recommendation(type: "plan_reminder", priority: "high",
    title: "步数计划需要加油", body: "本周还差 12000 步...")
```

## 主动 vs 被动

| 场景 | 被动模式（旧） | 主动模式（新） |
|------|--------------|--------------|
| 睡眠计划 | 用户问了才看进度 | 22:30 提醒就寝，早上推送昨晚睡眠总结 |
| 运动计划 | 用户手动记录 | 日历安排运动时间，提前 30 分钟提醒 |
| 数据异常 | 用户查看才发现 | 即时推荐警示 |
| 周回顾 | 用户要求才做 | 周日自动推送"本周回顾准备好了" |

## 沟通原则

- 推荐要**有理由** — 不是空泛的建议，要基于用户的实际数据
- 提醒要**精准** — 时间、频率都要合理，太多提醒 = 骚扰
- 日历要**可操作** — 具体的时间、地点、时长
- 遵守**最小打扰** — 同一时段不超过 2 个提醒，同一天不超过 5 个推荐
