# PHA 项目记忆系统原理分析

## 1. 系统架构

### 1.1 核心组件

| 文件 | 功能1 |
|------|------|
| **memory-manager.ts** | 记忆管理主入口，负责协调所有记忆操作 |
| **vector-store.ts** | 向量存储，使用 sqlite-vec 和 OpenAI Embedding |
| **embeddings.ts** | Embedding 提供者，调用 OpenRouter API |
| **profile.ts** | 用户档案管理（PROFILE.md）和记忆文件操作（MEMORY.md、daily logs） |
| **schema.ts** | SQLite 数据库 schema 定义（chunks、vec_chunks 表） |
| **info-collector.ts** | 信息收集器，从对话中提取用户档案信息 |
| **hybrid.ts** | 混合搜索算法，合并向量搜索和关键字搜索结果 |

### 1.2 数据存储策略

记忆系统采用**双重存储**策略：

#### 文件系统存储（`.pha/users/<uuid>/` 目录）
```
.pha/users/<uuid>/
├── PROFILE.md              # 用户健康档案
├── MEMORY.md               # 长期记忆总结
├── memory/
│   └── 2024-01-15.md       # 每日对话记录
└── sessions/
    └── <sessionId>.jsonl   # 会话转录
```

#### SQLite 数据库存储（`.pha/memory.db`）
- `chunks` 表：存储分块后的文本及元数据
- `chunks_fts`：FTS5 全文搜索索引
- `vec_chunks`：sqlite-vec 向量存储（嵌入向量）
- `users` 表：用户信息

## 2. 记忆检索方式

### 2.1 混合搜索策略（推荐）

`MemoryManager.searchAsync()` 方法实现了**混合搜索**：

```typescript
// 同时执行向量搜索和关键字搜索
const [vectorResults, keywordResults] = await Promise.all([
  this.vectorSearchInternal(uuid, query, candidates),
  Promise.resolve(this.keywordSearchInternal(uuid, query, candidates)),
]);

// 合并结果（使用权重配置）
const merged = mergeHybridResults({
  vector: vectorResults,
  keyword: keywordResults,
  vectorWeight: 0.7,    // 向量搜索权重
  textWeight: 0.3,      // 关键字搜索权重
});
```

### 2.2 搜索模式对比

| 模式 | 适用场景 | 实现方法 |
|------|----------|----------|
| **混合搜索**（默认） | 平衡语义理解和精确匹配 | `searchAsync()` |
| **向量搜索** | 语义相似度搜索 | `vectorStore.search()` |
| **关键字搜索** | 精确匹配搜索 | `keywordSearchInternal()` |
| **简单搜索** | 备用方案（无 FTS 时） | `simpleSearch()` |

## 3. 记忆使用场景

### 3.1 会话记忆管理

```typescript
// 保存重要信息到长期记忆
memoryManager.appendMemory(uuid, "用户提到有高血压病史");

// 保存每日对话记录
memoryManager.appendDailyLog(uuid, "今日讨论了高血压用药问题");

// 保存会话转录
memoryManager.appendSessionTranscript(uuid, sessionId, transcript);
```

### 3.2 记忆检索工具

项目提供了三个 MCP 工具供 Agent 使用：

| 工具 | 功能 | 参数 |
|------|------|------|
| `memory_search` | 搜索用户记忆 | `query`, `maxResults` |
| `memory_save` | 保存重要信息到长期记忆 | `content` |
| `daily_log` | 记录每日对话亮点 | `content` |

### 3.3 系统提示构建

## 4. 配置与依赖

### 4.1 Embedding 配置（`.pha/config.json`）

```jsonc
{
  "embedding": {
    "enabled": true,          // 是否启用向量搜索
    "model": "openai/text-embedding-3-small" // Embedding 模型
  },
  "llm": {
    "provider": "openrouter", // 使用 OpenRouter 作为 Embedding 提供者
    "apiKey": "sk-or-v1-xxx"
  }
}
```

### 4.2 依赖库

- `sqlite-vec`：SQLite 向量扩展
- `openai`：Embedding API（通过 OpenRouter）
- `bun:sqlite`：Bun 内置 SQLite 驱动

## 5. 系统特点

1. **智能分块**：文本自动分块，保留上下文重叠
2. **混合检索**：结合语义搜索和关键字搜索的优势
3. **增量索引**：记忆内容实时索引，支持快速检索
4. **多用户支持**：每个用户有独立的记忆空间
5. **容错设计**：无 Embedding 时自动降级到 FTS 搜索
